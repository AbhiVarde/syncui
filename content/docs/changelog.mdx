##### v1.2.0 – MCP

**New:**  
→ **MCP Server** – Let your AI agent add components directly into your project  
→ Published on npm: [@abhivarde/syncui-mcp](https://www.npmjs.com/package/@abhivarde/syncui-mcp)

**Setup:**  
→ Add to your MCP client config (Claude Code, Cursor, Windsurf, Codex, OpenCode)

```json
{
  "mcpServers": {
    "syncui": {
      "command": "npx",
      "args": ["-y", "@abhivarde/syncui-mcp"]
    }
  }
}
```

**Why it matters:**  
→ No commands, no docs tab  
→ Just tell your agent which component you want  
→ You still own the code

##### v1.1.0 – CLI

**New:**  
→ **CLI** – Add any component or block directly into your project  
→ Published on npm: [@abhivarde/syncui](https://www.npmjs.com/package/@abhivarde/syncui)

<PackageManagerTabs
  npm="npx @abhivarde/syncui@latest add name/variant"
  yarn="yarn dlx @abhivarde/syncui@latest add name/variant"
  pnpm="pnpm dlx @abhivarde/syncui@latest add name/variant"
  bun="bunx --bun @abhivarde/syncui@latest add name/variant"
/>

Use `name/variant` to target exactly what you need, e.g. `accordion/brutalist` or `hero/centered`.

**Why it matters:**  
→ No copy-paste, no docs tab  
→ One command, straight into your codebase

→ 🚀 Smaller bundles & faster load times

→ 🧠 Cleaner architecture with fewer abstractions  
→ ⚡ Noticeably faster UI interactions
